# EDA Prompt 回帰チェックリスト

Prompt を修正した場合や、モデルを変更した場合は、
`tests/titanic_summary.txt` で再実行し、以下を確認する。

---

## A. 構造チェック

- [ ] 指定された見出し順を守っている
- [ ] 最大点数制限（例：最大5点など）を守っている
- [ ] 不要な冗長出力がない

---

## B. 事実の厳密性

- [ ] describe() から導けない内容を断定していない
- [ ] 推測は明示されている
- [ ] 外部知識（タイタニックの一般知識等）を混入していない

---

## C. 欠損処理

- [ ] 欠損率が明示的に算出されている
- [ ] 欠損率10%以上の変数が列挙されている
- [ ] 欠損リスクが1文で明示されている

---

## D. 外れ値 / レア値処理

- [ ] Continuous のみに IQR を適用している
- [ ] Count/Binary/Ordinal に IQR を誤適用していない
- [ ] IQR=0 等の不安定条件を明示している
- [ ] 「データエラーの可能性」と「レアだが真実の可能性」を分けている
- [ ] 削除禁止などのデフォルト方針を守っている（該当バージョン）

---

## E. 言語制約

- [ ] 禁止された曖昧語を使用していない
- [ ] 確定不能な点は「統計量だけでは確定できない」と明示している

---

## F. 実験設計

- [ ] 実験は3つに限定されている
- [ ] 各実験に「目的」「手法」「評価指標」がある
- [ ] 成功判定が定量化されている（該当バージョン）

---

## G. 重大な回帰兆候

- [ ] 構造が崩れている
- [ ] 外れ値ロジックが簡略化されている
- [ ] 推測が事実扱いになっている
- [ ] 出力が抽象的になりすぎている
